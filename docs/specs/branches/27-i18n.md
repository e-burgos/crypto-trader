# Spec 27 — Internacionalización (i18n) EN/ES

**Fecha:** 2026-04-12
**Versión:** 1.0
**Estado:** Implementado (retrospec)
**Branch:** `feature/i18n`
**Dependencias:** Spec 07 (frontend-foundation), Spec 16 (frontend-completion)

---

## 1. Resumen ejecutivo

El spec original no incluía soporte de múltiples idiomas. La implementación añadió un sistema de internacionalización completo basado en **react-i18next** con soporte para:

- **Español** (ES) — idioma primario
- **Inglés** (EN) — idioma secundario

El idioma se detecta automáticamente del navegador y puede cambiarse desde el header. ~1400 claves de traducción cubren todas las páginas del dashboard.

## 3. Configuración

### 3.1 Archivo de configuración

`apps/web/src/lib/i18n.ts` (o `i18n.config.ts`):

```typescript
import i18n from 'i18next';
import { initReactI18next } from 'react-i18next';
import LanguageDetector from 'i18next-browser-languagedetector';
import en from '../locales/en';
import es from '../locales/es';

i18n
  .use(LanguageDetector)
  .use(initReactI18next)
  .init({
    resources: {
      en: { translation: en },
      es: { translation: es },
    },
    fallbackLng: 'es',
    interpolation: { escapeValue: false },
    detection: {
      order: ['localStorage', 'navigator'],
      caches: ['localStorage'],
    },
  });

export default i18n;
```

### 3.2 Integración en React

En `main.tsx`:
```typescript
import './lib/i18n'; // side-effect: inicializa i18next

// En el componente App:
const { i18n } = useTranslation();
// Cambio de idioma: i18n.changeLanguage('en' | 'es')
```

### 4.2 Convenciones de claves

- `camelCase` para todas las claves

- Sufijo `Placeholder` para placeholders
- Sufijo `Toast` para mensajes de toast
- Interpolación con `{{variable}}` para valores dinámicos

## 5. Selector de idioma (UI)

### 5.1 Ubicación

El selector de idioma está en el **header del dashboard** y en el **navbar de la landing page**:
- Botón compacto con la bandera del idioma actual
- Dropdown con 2 opciones: 🇦🇷 Español | 🇺🇸 English

### 5.2 Persistencia

El idioma seleccionado se persiste en `localStorage` con la clave `i18nextLng`. Al recargar la página, se restaura el idioma elegido.

### 5.3 Cambio de idioma sin reload

`i18n.changeLanguage('en')` aplica el cambio inmediatamente en todos los componentes que usan `useTranslation()` sin recargar la página (react-i18next maneja la re-renderización).

## 6. Uso en componentes

### Patrón estándar

```typescript
import { useTranslation } from 'react-i18next';

function MyComponent() {
  const { t } = useTranslation();
  
  return (
    <div>
      <h1>{t('config.title')}</h1>
      <p>{t('config.guide.sourceNewsSub', { count: 5 })}</p>
    </div>
  );
}
```

### Plurales

```typescript
// En el archivo de traducción:
const es = {
  trades: '{{count}} trade',
  trades_other: '{{count}} trades',
};

// En el componente:
t('trades', { count: totalTrades })
```

## 7. Cobertura de traducción

### Páginas 100% traducidas

- Landing page (todas las secciones)
- Auth (login, registro)
- Onboarding wizard (todos los pasos)
- Dashboard overview
- Live chart
- Trade history (tabla + modal de detalle)
- Agent log
- Bot analysis
- Positions (tabla + modal de detalle)
- Config (formulario completo + presets + guías)
- Settings (todos los forms de API keys)
- News feed
- Analytics
- Notifications
- Chat (KRYPTO)
- Admin panel
- Help page

### Strings no traducidos (excluidos por diseño)

- Nombres de providers LLM (Claude, OpenAI, Groq) — son nombres propios
- Símbolos de assets (BTC, ETH, USDT, USDC)
- Etiquetas de indicadores técnicos (RSI, MACD, EMA) — acrónimos técnicos universales
- Nombres de fuentes de noticias (CoinDesk, CoinTelegraph)
- Valores numéricos y porcentajes

## 8. Criterios de aceptación

- [ ] El selector de idioma en el header cambia el idioma inmediatamente
- [ ] El idioma se persiste en localStorage y sobrevive recargas
- [ ] Todas las páginas del dashboard están 100% traducidas al español
- [ ] Todas las páginas del dashboard están 100% traducidas al inglés
- [ ] Los mensajes con variables dinámicas (`{{count}}`, `{{name}}`) se interpolan correctamente
- [ ] No hay strings hardcodeados en inglés en ningún componente del dashboard
- [ ] El fallback es español cuando una clave no existe en el idioma seleccionado
- [ ] Los toasts de éxito/error respetan el idioma activo
